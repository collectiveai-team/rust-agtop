# rust-agtop ‚Äî Best-Practices Audit Report

**Date:** 2026-04-17
**Scope:** read-only audit of 8.5K LOC across `agtop-core` + `agtop-cli` against
Rust and ratatui best practices.
**Method:** 3 parallel subagent code reviews + `cargo fmt --check`,
`cargo clippy -- -D warnings`, `cargo clippy -W clippy::pedantic`, `cargo doc`,
`cargo tree`.
**No files were modified during this audit.**

## Overall verdict

The code is **well-written**. Disciplined App/render separation, thiserror+anyhow
split done correctly, rustls-only HTTP, `TestBackend` for TUI tests, panic hook
installed. The biggest problems are **infrastructural** (no CI, no LICENSE, no
lint policy) and a handful of **architectural smells** (global pricing state,
1000+ LOC files with near-duplicated parse logic, `f64` money, one dead widget
file).

## Recommended execution order

1. **Week 1 ‚Äî infrastructure (low risk, high payoff):** items 1, 2, 3, 5, 6 +
   fix the broken doc link + delete `.ruff_cache/`. Each ships as a tiny,
   reviewable PR.
2. **Week 2 ‚Äî API stability + error handling:** items 4, 8, plus M4 / M8 / M9 /
   M10. These are the "before 0.2 public release" changes.
3. **Week 3 ‚Äî TUI correctness + perf:** items 10, 11, 12, 13, 14 + M24.
4. **Week 4 ‚Äî structural refactor:** items 7, 9, 15, plus splitting `app.rs`
   (M1) and the 3 large provider files.
5. **Later / optional:** `f64 ‚Üí Decimal` money migration (H4), token newtype
   (H5), `tokio ‚Üí std::thread` (M6), `chrono ‚Üí jiff`, property tests.

Total delta at completion: ~400‚Äì600 LOC removed (via de-duplication), ~250 LOC
of CI/config added, ~500 LOC of new tests, and the library reaches a shape
that's publishable to crates.io without breaking-change regret.
