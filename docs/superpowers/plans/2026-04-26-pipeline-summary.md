## Open follow-ups

These items were not fully implemented and are candidates for a follow-up PR:

### FU-1: Parser → SessionState wiring (AC4)
All 7 parsers (`claude`, `codex`, `opencode`, `copilot`, `cursor`, `gemini_cli`, `antigravity`) set `session_state: None` and rely on `state_resolution::resolve_state` for derivation. The spec requires parsers to produce `SessionState` variants directly. A follow-up should wire the state derivation into each parser's output or update the spec to accept the current delegation approach.

### FU-2: Stalled threshold runtime wiring (AC7)
`resolve_state_with_threshold(Duration)` exists in core, and `RefreshModel::stalled_threshold_secs` is in the config. However, the v2 app layer does not read the persisted config value and pass it into state resolution. Follow-up: wire `config.refresh.stalled_threshold_secs` → `resolve_state_with_threshold` in the refresh/update cycle.

### FU-3: NO_COLOR → animations_enabled + frame budget auto-disable (AC8)
When `NO_COLOR` is set, colour depth is reduced to Ansi16 but `animations_enabled` is not set to false. Additionally, no frame-budget mechanism exists to auto-disable pulsation when rendering is too slow. Follow-up: (a) read `NO_COLOR` env var during startup and set `animations_enabled = false`; (b) implement a simple frame timing guard.

### FU-4: Dual-flavor snapshot tests (AC18)
Only the Appearance config section runs snapshots in both `nf_off` and `nf_on` modes. Sessions table, aggregation view, header, quota, tab-bar, and info drawer snapshots need a second `nf_on` test fixture. Follow-up: parameterize all snapshot tests with both values.

### FU-5: Shift+click bypass (AC19)
The sessions table mouse handler does not check for `KeyModifiers::SHIFT` on mouse-down events. Follow-up: detect Shift modifier and pass the click through to the terminal (bypass mouse capture) per the spec.

### FU-6: Missing snapshot sizes (AC20)
The spec mandates snapshots at (140×40) and narrow-mode breakpoints (80×24, 100×30). Current tests use 140×12, 140×20, 140×30, 140×3, 80×3. Follow-up: add the three missing size variants to all component snapshot suites.
